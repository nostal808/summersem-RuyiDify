# 知识库 03：元数据过滤

更新时间：2026-07-09

## PPT 可用标题

元数据过滤：让知识库只在正确范围内检索

## 一句话讲清楚

默认情况下，检索会覆盖整个知识库；启用元数据过滤后，可以把检索限制在符合条件的文档范围内，从而提高大知识库和多场景知识库的精度。

## 为什么需要元数据过滤

真实知识库经常不是一份文档，而是一堆资料：

- 不同课程章节。
- 不同产品版本。
- 不同业务部门。
- 不同客户项目。
- 不同发布时间的制度或手册。

如果不做过滤，用户问一个具体问题时，系统可能从不相关范围里召回内容。

## 适合 PPT 的例子

一个知识库里同时放了这些资料：

- `课程 = Dify 知识库入门`
- `课程 = Dify 工作流开发`
- `版本 = 2025`
- `版本 = 2026`
- `类型 = 课件`
- `类型 = FAQ`

用户问：“创建通用型知识库的第一步是什么？”

更合理的检索范围是：

- `课程 = Dify 知识库入门`
- `版本 = 2026`
- `类型 = 课件`

## 可直接放 PPT 的流程

```mermaid
flowchart LR
    A["用户问题"] --> B["读取元数据条件"]
    B --> C["筛选候选文档"]
    C --> D["在候选范围内检索"]
    D --> E["返回更精准的 chunks"]
```

## 讲课重点

- 元数据过滤不是搜索技巧，而是知识库治理能力。
- 文档越多、主题越杂，元数据越重要。
- 元数据可以按课程、章节、版本、客户、业务线、文档类型设计。
- 没有元数据的大知识库，后期很容易变成资料杂货铺。

## 适合 RuyiDify 的元数据字段

| 字段 | 示例 | 用途 |
| --- | --- | --- |
| course | Dify 知识库开发入门 | 区分课程 |
| chapter | 06 | 区分章节 |
| version | 2026 | 区分版本 |
| doc_type | ppt / faq / transcript | 区分资料类型 |
| topic | knowledge-base | 区分主题 |

## 可直接放 PPT 的金句

知识库越大，越不能只靠相似度检索；先用元数据限定范围，再做语义检索，答案才更稳。
