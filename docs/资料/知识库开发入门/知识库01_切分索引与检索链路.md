# 知识库 01：切分、索引与检索链路

更新时间：2026-07-09

## PPT 可用标题

Dify 知识库不是文件夹，而是一条 RAG 工程链路

## 一句话讲清楚

创建知识库只是开始，真正决定问答质量的是资料如何被处理、切分、索引、检索、重排，并最终作为上下文交给大模型。

## 课程讲解顺序

1. 原始资料进入知识库。
2. Dify 对资料进行处理和切分，形成 chunks。
3. chunks 被向量化并建立索引。
4. 用户提出问题后，系统根据 query 检索相关 chunks。
5. 检索结果经过过滤或重排。
6. 最终结果进入 LLM 上下文。
7. LLM 基于上下文生成回答。

## 适合画成 PPT 的流程图

```mermaid
flowchart LR
    A["原始资料"] --> B["文档处理"]
    B --> C["切分 chunks"]
    C --> D["向量化与索引"]
    D --> E["用户问题"]
    E --> F["知识检索"]
    F --> G["过滤与重排"]
    G --> H["注入上下文"]
    H --> I["LLM 回答"]
```

## 讲课重点

- 知识库不是上传区，而是上下文生产系统。
- 文档切分会影响召回质量。
- 索引质量会影响能否找回相关内容。
- 检索结果不一定都该进入模型，还需要 Top K、阈值、重排和过滤。
- 回答不准时，不要只怪模型，要排查资料、切分、召回、重排和提示词。

## 可直接放 PPT 的金句

知识库问答的质量，不是从模型回答时才开始决定的，而是从资料被切成第一个 chunk 时就已经开始决定了。

## 课堂演示建议

- 准备一份短文档，先默认切分并测试问题。
- 再换一种切分方式，对比召回片段。
- 让学员看到同一个问题为什么会因为 chunk 不同而出现不同答案。
