# 知识库 05：PPT 页面规划

更新时间：2026-07-09

## 用途

这份文件用于把知识库专题资料转成 PPT 页面。原则是：先做 PPT，再根据 PPT 写博客。

## 建议 PPT 结构

### 第 1 页：本节目标

标题：这一节我们学会创建并理解 Dify 通用型知识库

要点：

- 创建通用型知识库。
- 创建空知识库。
- 理解知识库不是文件夹。
- 为后续上传资料、检索测试、接入应用做准备。

### 第 2 页：知识库不是上传文档

标题：知识库不是上传区，而是上下文生产系统

要点：

- 上传文档只是第一步。
- 切分、索引、检索、重排才决定回答质量。
- 回答不准时，要排查整条 RAG 链路。

### 第 3 页：创建通用型知识库

标题：选择创建通用型知识库，先跑通主线

配图：

- `docs/博客/images/004-01-create-general-knowledge.png`

讲解：

- 通用型知识库适合入门教学和业务演示。
- 先理解 Dify 内置知识库，再扩展外部知识库。

### 第 4 页：创建空知识库

标题：先创建空知识库，再让资料有地方生长

配图：

- `docs/博客/images/004-02-create-empty-knowledge.png`

讲解：

- 空库代表先定边界。
- 不要一开始就把所有资料混在一起。

### 第 5 页：输入知识库名称

标题：知识库命名要表达业务边界

配图：

- `docs/博客/images/004-03-input-knowledge-name.png`

讲解：

- 不要用“测试库”“资料库”这类泛名。
- 推荐按课程、业务、版本命名。

### 第 6 页：创建完成

标题：创建完成只是地基，不是知识库已经可用了

配图：

- `docs/博客/images/004-04-knowledge-created.png`

讲解：

- 下一步还要上传资料、等待索引、做检索测试。
- 知识库要接入应用或工作流才形成业务闭环。

### 第 7 页：Top K、阈值和 Rerank

标题：三个参数决定哪些证据进入上下文

要点：

- Top K 控制返回数量。
- Score Threshold 控制最低相似度。
- Rerank 控制候选结果排序。

### 第 8 页：元数据过滤

标题：知识库越大，越需要先限定检索范围

要点：

- 按课程、章节、版本、类型做元数据。
- 先过滤范围，再做语义检索。
- 避免旧资料和无关资料混入上下文。

### 第 9 页：外部知识库 API

标题：已有 RAG 系统时，用外部知识库接入 Dify

要点：

- Dify 调用 `/retrieval`。
- 外部服务返回 records。
- 适合企业已有向量库或搜索系统的场景。

### 第 10 页：课堂练习

标题：创建一个课程资料知识库

练习任务：

1. 创建通用型空知识库。
2. 使用清晰的业务命名。
3. 上传一份课程资料。
4. 提出 3 个测试问题。
5. 记录召回结果和回答质量。
