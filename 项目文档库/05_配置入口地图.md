## 5.1 配置层级

| 层级 | 文件 | 作用域 | 备注 |
|------|------|--------|------|
| Docker 主 | `docker/.env` ← `docker/.env.example` | 主 compose 全部服务 | 最常用入口 |
| Docker 主题 | `docker/envs/*.env`（core-services、databases、vectorstores、infrastructure、security） | 按主题拆分的高级项 | README:140 |
| Docker 中间件 | `docker/middleware.env` ← `docker/envs/middleware.env.example` | 仅依赖（middleware compose） | Makefile prepare-docker |
| 后端本地 | `api/.env` ← `api/.env.example` | 本地 uv 起 api 时 | 读码经 `configs.dify_config` |
| 前端本地 | `web/.env.local` ← `web/.env.example` | 本地 pnpm 起 web 时 | |
| 代码内 | `configs/dify_config`（api） | 统一配置访问 | `api/AGENTS.md:206` 禁止直接读 env |

## 5.2 本地开发配置入口

- 后端：`api/.env`（由 `api/.env.example` 复制）。关键项：DB 连接、Redis、向量库、模型/Plugin、存储、Secret。
- 前端：`web/.env.local`（由 `web/.env.example` 复制）。关键项：`CONSOLE_API_URL`、`APP_API_URL`、`NEXT_PUBLIC_*`、功能开关。
- 依赖：`docker/middleware.env`（由 `docker/envs/middleware.env.example` 复制），供 middleware compose 使用。

## 5.3 模型与 Embedding 配置

- 模型供应商主要通过 **Plugin Daemon** 安装/配置（控制台 UI 或 API），运行时经 `PLUGIN_DAEMON_URL`/`PLUGIN_DAEMON_KEY` 通信。
- 内置 provider（如 OpenAI）示例变量：`OPENAI_API_BASE`（`.env.example:38`，默认 `https://api.openai.com/v1`）。⚠️ 真实 key 不应硬编码，应走 Plugin/密钥管理。
- Embedding 模型：用于高质索引（`indexing_technique=high_quality`）。在知识库设置中选择 embedding 模型；底层经 `core/model_manager` + Plugin provider 调用。
- Rerank 模型：检索重排，配置于知识库 `retrieval_model`（examples/18、docs 02）。

## 5.4 向量数据库配置（按 `VECTOR_STORE` 选）

- 主开关：`VECTOR_STORE`（`.env.example:164`，默认 `weaviate`）。
- 启用对应 compose **profile** 启动该向量库容器（见 02/04）。
- 各向量库独立 env 段（`docker/.env.example` 与 `docker/envs/vectorstores/*.env`），例如：
  - Weaviate：`WEAVIATE_ENDPOINT`、`WEAVIATE_API_KEY`、`WEAVIATE_GRPC_ENDPOINT`、`WEAVIATE_TOKENIZATION`（`.env.example:166-183`）
  - Qdrant：`QDRANT_API_KEY`（compose qdrant 服务）
  - pgvector：`PGVECTOR_POSTGRES_PASSWORD`、`PGVECTOR_POSTGRES_DB`（compose pgvector 服务）
  - Chroma / Milvus / OceanBase / 等等：各自 env 段 + profile。
- 支持的向量库完整列表见 `api/providers/vdb/*`（pyproject `vdb-all` 组，约 27 个）。

## 5.5 API Key / 密钥配置

| 变量（示例/位置） | 作用 | 注意 |
|------------------|------|------|
| `SECRET_KEY` (`.env.example:34`) | Flask 会话签名密钥 | 必须强随机 |
| `INIT_PASSWORD` (`.env.example:35`) | 初始化管理员密码 | 首次安装 |
| `PLUGIN_DAEMON_KEY` (`.env.example:219`) | Plugin Daemon 通信密钥 | 示例值需替换 |
| `PLUGIN_DIFY_INNER_API_KEY` (`.env.example:228`) | API↔Plugin 内部调用 | 示例值需替换 |
| `INNER_API_KEY_FOR_PLUGIN` (compose api/worker environment) | 同上（compose 内） | 示例值需替换 |
| `WEAVIATE_API_KEY` / `QDRANT_API_KEY` 等 | 向量库访问 | 示例值需替换 |
| `REDIS_PASSWORD` / `DB_PASSWORD` | 中间件密码 | 默认 `difyai123456` 仅示例 |
| 模型供应商 Key | 经 Plugin/密钥管理，不全在 .env | 禁止明文提交 |

## 5.6 存储配置

- `STORAGE_TYPE=opendal`（`.env.example:161`），支持本地/S3/OSS/MinIO 等；通过 `PLUGIN_STORAGE_*` 与对象存储桶相关变量配置（compose plugin_daemon 段有大量 `PLUGIN_STORAGE_*`/`AWS_*`/`ALIYUN_OSS_*`/`TENCENT_COS_*`/`VOLCENGINE_TOS_*`）。

## 5.7 前端相关配置

- `CONSOLE_API_URL` / `APP_API_URL`（`.env.example:16,21`）：前端调用的 API 地址。
- `TOP_K_MAX_VALUE`（compose web 默认 10）、`INDEXING_MAX_SEGMENTATION_TOKENS_LENGTH`（默认 4000）等检索/索引上限。
- `EXPERIMENTAL_ENABLE_VINEXT`、`ENABLE_WEBSITE_JINAREADER/FIRECRAWL/WATERCRAWL` 等功能开关。
