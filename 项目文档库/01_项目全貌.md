## 1.1 RuyiDify 与开源 Dify 的关系

- **RuyiDify 是 Dify 的衍生/教学分支**，不是独立自研产品。它直接复用 Dify 的上游代码（api/web/docker 等），版本基座为 `dify-api==1.15.0`（`api/pyproject.toml:3`），docker 镜像 `langgenius/dify-api:1.15.0` / `langgenius/dify-web:1.15.0`（`docker/docker-compose.yaml`）。
- **定位差异（关键）：** 根据根 `AGENTS.md`，RuyiDify 被做成一门「**实战课**」（面向想真正理解并二开 Dify 的开发者）。作者在 `docs\博客\` 下以学生视角写实战文章（参考 `docs\博客\001_我重新看了一遍Dify，决定把RuyiDify做成一门实战课.md`）。
- **非上游内容（本项目新增、随仓库但归作者维护）：**
  - `docs\博客\` — 实战课文章（公开，第一人称，张大鹏视角）。
  - `docs\资料\` — 专题学习笔记（如 `docs\资料\知识库开发入门\`）。
  - `examples\知识库\` — 28+ 个可运行的中文注释 Python 示例，演示知识库 API 全链路。
- **上游 Dify 原生能力：** Workflow、RAG Pipeline、Agent、模型管理、LLMOps、BaaS API、20+ 向量库支持、Plugin 体系。

## 1.2 技术栈

| 层 | 技术 | 证据 |
|----|------|------|
| 前端 Web | Next.js + React + TypeScript（pnpm workspace，含 `packages/dify-ui`、`packages/iconify-collections`） | `web/package.json` scripts、`pnpm-workspace.yaml` |
| 后端 API | Python 3.12 + Flask + Flask-RESTX + gunicorn + gevent | `api/pyproject.toml` requires-python `~=3.12.0`、依赖 flask/gunicorn/gevent |
| 异步任务 | Celery + Redis（broker/result） | `api/celery_entrypoint.py`、`docker-compose` worker 服务 |
| 关系库 | PostgreSQL 15（默认）/ MySQL 8（可选） | `docker-compose.yaml` db_postgres/db_mysql、`DB_TYPE` |
| 向量库 | Weaviate（默认）或 20+ 可选（Qdrant/Chroma/pgvector/Milvus/…） | `docker/.env.example:164 VECTOR_STORE=weaviate`、`api/providers/vdb/*` |
| 对象存储 | OpenDAL 抽象，支持 S3/OSS/MinIO/本地等 | `docker/.env.example:161 STORAGE_TYPE=opendal` |
| 模型供应商 | Plugin Daemon 体系 + 内置 providers（OpenAI 等） | `docker-compose.yaml` plugin_daemon 服务、`core/plugin` |
| 沙箱 | DifySandbox（代码执行隔离） | `docker-compose.middleware.yaml` sandbox 服务 |
| 编排 | Docker Compose（generated）+ Makefile + uv + pnpm | 见 `04_启动方式报告.md` |

## 1.4 Dify 原生能力 vs RuyiDify 扩展 vs 学习示例

- **Dify 原生能力：** 全部 `api/`、`web/`、`docker/`、Plugin 体系、Workflow/Agent/RAG 引擎、20+ 向量库、多模型供应商。
- **RuyiDify 扩展内容：** `docs\博客\`（实战课文章）、`docs\资料\知识库开发入门\`（专题笔记）、`examples\知识库\`（API 演示）。**代码层面尚未发现 fork 自有的业务二开目录**——扩展主要体现在「文档 + 示例 + 课程定位」。
- **学习示例：** `examples/知识库/01~28,32` 演示知识库 API；`docs\资料\知识库开发入门\` 提供理论 + 规划（见 `06_学习资源分类.md`）。

## 1.5 关键开发约定（来自 AGENTS.md，影响后续所有改动）

- 后端命令统一 `uv run --project api <command>`（根 `AGENTS.md:17`）。
- 配置只读 `configs.dify_config`，**禁止直接读环境变量**（`api/AGENTS.md:206`）。
- 异步任务经 `services/async_workflow_service`，任务实现于 `tasks/` 并显式选队列（`api/AGENTS.md:208`）。
- 前端用户字符串走 `web/i18n/en-US/`，禁止硬编码（`api/AGENTS.md:59`、`web/AGENTS.md:9`）。
- 不应以 agent 身份启动长服务（`uv run app.py`/`flask run` 等），但允许跑测试（`api/AGENTS.md:102`）。
- 博客发布在 `docs\博客\`，不提交/推送除非用户明确要求（`AGENTS.md:53`）。
