# 实习日志 · 孙达（U202360127 · 计236）

## 2026-07-15 实习记录

**实习进展**
今日在 RuyiDify（基于 Dify 1.15 自托管）项目上完成知识库接口梳理与端到端验证。先依据 Dify 官方“知识库 API”概览，整理出《Dify 知识库接口文档》，共 46 个接口、分 7 大类（知识库6/文档12/分段9/元数据7/标签7/模型1/知识流水线4）。随后通过数据集 API 创建“零基础AI编程”知识库，并上传该文档，索引状态达到 completed（48 个分段，由本地 Ollama 的 nomic-embed-text 嵌入）。最后调用 /retrieve 检索接口提问“Dify 知识库有多少个接口”，召回内容明确给出“46 个”，验证闭环成功。

**遇到的问题与解决**
1. OpenAI 在 Docker 容器内不可达（容器不继承宿主机 VPN 路由），保存密钥超时。改用本地 Ollama + 国内厂商（Tongyi/Qwen）绕过。
2. 插件安装卡在 PyPI 依赖拉取，网络恢复后顺利装好 Ollama 与通义插件。
3. 脚本硬编码端口 localhost:12010 不适用，实际 API 经 nginx 代理为 http://localhost/v1。

**新想法**
本地 embedding 免密钥、离线、可复现，更适合教学场景；云端 LLM 仅在做生成式问答时才需要密钥。

**实习收获**
打通了“建库→上传→索引→检索”全链路，理解了 Dify 插件机制、向量库与模型供应商协作，并产出可复用的项目文档库。
