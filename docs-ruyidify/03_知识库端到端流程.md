# 03 知识库端到端流程（真实代码路径）

- last_verified: 2026-07-15
- scope: `api/controllers/console/datasets/*`、`api/controllers/service_api/dataset/dataset.py`、`api/services/dataset_service.py`、`api/core/indexing_runner.py`、`api/tasks/document_indexing_task.py`、`api/core/rag/datasource/retrieval_service.py`、`api/services/knowledge_retrieval_inner_service.py`、`examples/知识库/*`
- evidence: ✅代码确认（符号/路径均来自读码）

> 全链路围绕 8 个核心概念：**Dataset（知识库）→ Document（文档）→ Segment（分段）→ Embedding（向量化）→ Index（索引）→ Retrieval（检索）→ Metadata（元数据过滤）→ 知识库问答（QA）**。

## 3.1 写路径：上传文档 → 向量入库

```
界面/API
  │  console: POST /console/api/datasets/{id}/documents  (datasets_document.py)
  │  service_api: POST /v1/datasets/{id}/document/create-by-file  (service_api/dataset/dataset.py)
  ▼
DatasetService / DocumentService
  │  services/dataset_service.py : save_document_with_dataset_id (L2133)
  │  创建 Document + DatasetProcessRule，落 PostgreSQL
  ▼
派发 Celery 任务
  │  tasks/document_indexing_task.py : document_indexing_task.delay(dataset_id, document_ids)  (L32, @shared_task(queue="dataset"))
  │  注：legacy 标记 TO BE DEPRECATED，新路径为 normal/priority_document_indexing_task（同文件推断）
  ▼
Worker 执行 _document_indexing (L49)
  │  取 Dataset/Document，校验 feature/额度
  ▼
IndexingRunner.run_in_indexing_status (core/indexing_runner.py:199)
  │  读 DocumentSegment（COMPLETED 的），转成 LlamaIndex Document（含 metadata: doc_id/doc_hash/document_id/dataset_id）
  │  父-子分片 (PARENT_CHILD_INDEX) 会带 child_chunks
  ▼
IndexProcessorFactory(index_type).init_index_processor()
  │  core/rag/index_processor/...  切分/清洗
  ▼
Embedding
  │  core/rag/embedding/embedding_base.py + cached_embedding.py  （经 ModelManager 调 embedding 模型）
  ▼
Vector（向量库写入）
  │  core/rag/datasource/vdb/vector_factory.py : Vector.add_texts (L224)
  │  → 具体后端（weaviate/qdrant/chroma/...）实现 VectorBase.add_texts (vdb/vector_base.py:31)
  ▼
PostgreSQL 记录 segment 状态 = COMPLETED；向量写入向量库
```

**关键文件/符号速查：**
- 控制器（写）：`api/controllers/console/datasets/datasets_document.py`、`api/controllers/service_api/dataset/dataset.py`
- 业务服务：`api/services/dataset_service.py`（`save_document_with_dataset_id` L2133）、`api/services/dataset_ref_service.py`
- 异步任务：`api/tasks/document_indexing_task.py`（`document_indexing_task` L32、`_document_indexing` L49）、`document_indexing_update_task.py`、`duplicate_document_indexing_task.py`
- 索引核心：`api/core/indexing_runner.py`（`run` L68、`run_in_indexing_status` L199）
- Embedding：`api/core/rag/embedding/embedding_base.py`、`cached_embedding.py`
- 向量写入：`api/core/rag/datasource/vdb/vector_factory.py`（`Vector.add_texts` L224）、`vdb/vector_base.py`（基类 L31）

## 3.2 读路径：提问 → 检索 → LLM 回答

```
用户问题 (query)
  │  应用 chat 接口 或  /v1/datasets/{id}/retrieve
  ▼
service_api: DatasetRetrieveApi.post  (service_api/dataset/dataset.py: 含 search_method 逻辑, L601 默认 keyword_search)
  │  或控制台/应用内部经 InnerKnowledgeRetrievalService.retrieve (services/knowledge_retrieval_inner_service.py:41)
  ▼
retrieval_service.retrieve (core/rag/datasource/retrieval_service.py)
  │  组装：向量检索 Vector.search_by_vector + 关键词检索 search_by_full_text (vector_factory.py L246/L270)
  │  + ExternalKnowledgeService（外部知识库模式）
  ▼
DataPostProcessor (core/rag/data_post_processor/data_post_processor.py)
  │  Reranking（RerankMode）、score_threshold、元数据过滤 (MetadataFilteringCondition)
  ▼
返回 RetrievalSegments / 命中分段（score、child_chunks）
  │  —— 若是「检索接口」则返回分段列表（examples/04）
  │  —— 若是「应用 chat」则继续 ↓
  ▼
LLM 节点：将命中上下文注入 prompt，经 Plugin Daemon / provider 生成回答（examples/32）
```

**检索参数（来自 examples 与 docs）：**
- `search_method`：`semantic_search` / `full_text_search` / `keyword_search` / `hybrid_search`（examples/16/17）
- `reranking_enable` + rerank 模型（examples/18、docs 02）
- `score_threshold_enabled` + `score_threshold`（examples/19、docs 02）
- `top_k`（examples/20、docs 02；web 侧 `TOP_K_MAX_VALUE` 默认 10，`docker-compose.yaml:405`）

## 3.3 元数据（Metadata）链路

- 字段管理：`examples/12_创建并删除元数据字段.py`、`14_启用和禁用内置元数据.py`
- 写入：`examples/13_给文档写入元数据.py`
- 过滤：`docs/资料/知识库开发入门/知识库03_元数据过滤.md`
- 实现位置推断：metadata 随 Segment 存入 PG（`DocumentSegment` 模型）并参与 `retrieval_service` 的 `MetadataFilteringCondition` 过滤。

## 3.4 RuyiDify 示例端口注意

`examples/知识库/*.py` 中 `API_BASE_URL = "http://localhost:12010/v1"`（如 `01_上传文档.py:15`）。
这说明作者演示时把 API 映射到了 **12010** 端口（非默认 5001）。读者本地需按自己端口修改 `API_BASE_URL`、`API_KEY`、`DATASET_ID`。
