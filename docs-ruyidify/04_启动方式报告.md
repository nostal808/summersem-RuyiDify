## 4.1 方式 A：Docker Compose 整体启动（📄文档说明 + ✅代码确认）

- **前置：** Docker + Docker Compose 已装（README:74）。
- **工作目录：** `docker/`
- **环境变量文件：** `docker/.env`（由 `.env.example` 复制；高级项在 `docker/envs/` 按主题拆分，README:140）。
- **准确命令：**
  ```bash
  cd docker
  cp .env.example .env        # 仅首次
  docker compose up -d
  ```
- **启动顺序（compose depends_on 控制）：** `init_permissions` → `db_postgres`/`db_mysql`/`redis` 健康 → `api`/`worker`/`worker_beat` → `plugin_daemon`/`sandbox`/`ssrf_proxy`/`weaviate` → `web` → `nginx`。⏳ 实际初始化顺序建议运行验证。
- **默认端口：** nginx 80/443、api 5001、plugin_daemon 5002/5003、sandbox 8194、redis 6379、postgres 5432、weaviate 8080。（来自 `docker-compose.yaml` 各 service `ports`/`environment`）
- **停止：** `docker compose down`（保留卷）；`docker compose down -v`（删卷）。
- **最小验证：** 浏览器打开 `http://localhost/install` 完成初始化（README:83）。
- **注意：** `docker-compose.yaml` 为 generated 文件，**不要手改**，改 `.env.example`/`docker-compose-template.yaml` 后重新生成（文件头注释）。

## 4.2 方式 B：本地源码开发（✅代码确认，来自 `Makefile dev-setup`）

- **前置：** Docker（仅依赖）、Python 3.12、uv、Node/pnpm、Git。
- **环境变量：** 自动从示例复制：`web/.env.local`（←`web/.env.example`）、`api/.env`（←`api/.env.example`）、`docker/middleware.env`（←`docker/envs/middleware.env.example`）。
- **准确命令（一步到位）：**
  ```bash
  make dev-setup
  ```
  其内部依次执行（`Makefile:18-46`）：
  1. `prepare-docker`：复制 `middleware.env`，`docker compose -f docker-compose.middleware.yaml --env-file middleware.env -p dify-middlewares-dev up -d`（起依赖）
  2. `prepare-web`：`cp -n web/.env.example web/.env.local`、`pnpm install`
  3. `prepare-api`：`cp -n api/.env.example api/.env`、`cd api && uv sync --dev`、`cd api && uv run flask db upgrade`
- **然后分别启动服务（读码推断，⏳需运行确认具体命令）：**
  - 前端：`cd web && pnpm dev`（web/package.json:29）→ `http://localhost:3000`
  - 后端：`cd api && uv run --project api flask run` 或 gunicorn（⏳ 最小验证：在 api 目录 `uv run flask run --help` 看可用；`api/AGENTS.md:102` 禁止 agent 长跑服务，但人可启）
  - Worker：`cd api && uv run celery -A celery_entrypoint.celery worker ...`（对齐 entrypoint.sh:66）
- **最小验证：** 访问 web 3000、api 5001/health（`docker-compose.yaml` api healthcheck: `curl -f http://localhost:5001/health`）。
- **停止：** `make dev-clean`（停 middleware compose + 删 volumes：`api/storage`、`docker/volumes/{db,mysql,redis,plugin_daemon,weaviate,sandbox/dependencies}`，Makefile:49-64）。

## 4.3 方式 C：前后端分别启动

- 前端：`cd web && pnpm dev`（dev）/ `pnpm build && pnpm start`（生产包）。`web/package.json` scripts（L25-49）。
- 后端：`cd api && uv run --project api <flask/gunicorn>`（需先 `uv sync --dev` 与 `flask db upgrade`，见方式 B 的 prepare-api）。
- 依赖（PG/Redis/向量库/Sandbox/Plugin）仍需方式 E 或 A 提供。

## 4.4 方式 D：Worker / Beat 单独启动

- 经 api 容器 `MODE` 分派（entrypoint.sh）：
  - Worker：`MODE=worker` → `celery -A celery_entrypoint.celery worker -P <pool> ...`（entrypoint.sh:21,66）
  - Beat：`MODE=beat` → `celery -A app.celery beat`（entrypoint.sh:71）
  - 本地等价：`cd api && uv run celery -A celery_entrypoint.celery worker`（⏳参数对齐需运行确认）。
- 适用：只处理索引/工作流队列，不启 API 服务。

## 4.5 方式 E：仅 Docker 中间件（依赖供给）

- 命令：`make prepare-docker` 或等价 `docker compose -f docker-compose.middleware.yaml --env-file middleware.env -p dify-middlewares-dev up -d`（`Makefile:30`）。
- 包含：db_postgres、db_mysql、redis、sandbox、plugin_daemon、ssrf_proxy、weaviate（见 `docker-compose.middleware.yaml`）。
- 用途：本地源码开发时只起依赖，api/web 在宿主机跑。

## 4.6 方式 F/G：构建与质量（非运行服务）

- 构建：`make build-api`（`docker build -f api/Dockerfile`）、`make build-web`（`docker build -f web/Dockerfile`）、`make push-*`、`make build-push-all`（`Makefile:152-184`）。
- 质量：`make format`/`check`/`lint`/`type-check`/`test`/`test-all`（见 `api/AGENTS.md:180-191`）。测试可用 `--start-middleware`/`--start-vdb` 自动起依赖（`Makefile:135,142`）。

## 4.8 推荐选择流程

1. 想**完整体验/类生产** → 方式 A（docker compose up -d）。
2. 想**改源码/二开调试** → 方式 E（起依赖）+ 方式 B/C（本地 api/web）。
3. 只**修索引/异步问题** → 方式 D（worker/beat）。
4. 只**出包/CI** → 方式 F/G。

## 4.9 风险与待验证项

- ⚠️ **Windows 阻塞点：** `Makefile` 用 bash（含 `$$`、`cp -n`、heredoc）。在 Windows 上需用 Git Bash / WSL；原生 cmd/PowerShell 不支持（README/AGENTS 未明确，推断⏳）。`docker-compose.middleware.yaml` 路径 `./volumes/...` 在 Windows Docker Desktop 需文件共享。
- ⚠️ **端口占用：** 默认 80/443/5001/5002/5432/6379/8080 若被占用会起不来；examples 用 12010，注意区分。
- ⚠️ **镜像下载：** `langgenius/dify-*`、`weaviate`、`pgvector`、`milvus` 等镜像较大，首次 `up` 耗时且需联网。
- ⚠️ **数据库迁移：** 本地源码方式需 `uv run flask db upgrade`（Makefile prepare-api），漏做会 500。
- ⚠️ **模型配置：** 高质索引需配置 Embedding 模型（Plugin Daemon 或内置 provider）；examples/04 用 `keyword_search` 规避，但正式链路必需 embedding（见 05）。
- ⏳ **未运行确认项：** api 本地具体启动命令（`flask run` vs gunicorn 参数）、compose 实际启动顺序耗时、Windows 下 Makefile 可用性、12010 端口映射来源（作者本地配置，非仓库默认）。
